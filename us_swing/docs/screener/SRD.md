## Section 3: Three-Stage Execution Pipeline (FO-SCR-003)

| ID | Parent | P | Description | In | Out | Constraints | Status |
|---|---|---|---|---|---|---|---|
| SRD-SCR-003.001 | FO-SCR-003 | Must | `PreFilter` class: Quick, single-threaded filter. Checks: price > $5, volume (from latest bar) > 1M, stock not halted. Returns filtered list of symbols. Time: <1s for 500 symbols (includes DB fetch). | 500 symbols, latest 1d bars | ~300 symbols | Hard filters; no scoring | Draft |
| SRD-SCR-003.002 | FO-SCR-003 | Must | `PresetExecutor.run_preset(preset_id, user_id, manual=False)` orchestrates 3-stage pipeline. Loads preset, validates permissions, fetches bars from DB, runs pre-filter, Stage 2 (parallel), Stage 3 (optional LLM), returns `ScreenerRunResult` with timestamp, execution_mode, results. | preset_id str, user_id str | `ScreenerRunResult` dict | Execution mode: "scheduled" or "manual" | Draft |
| SRD-SCR-003.003 | FO-SCR-003 | Must | **Stage 2: Parallel Screener Execution.** Instantiate screeners from ScreenerRegistry per preset config. CPU-bound screeners (indicator, ML) run in multiprocessing.Pool (max_workers=4). I/O-bound screeners (LLM for filtering, MCP) run in asyncio (if any; else skip asyncio). Collect results per screener. | 300 symbols, screener list | per-screener results dict | No dependencies between screeners; safe to parallelize | Draft |
| SRD-SCR-003.004 | FO-SCR-003 | Must | **Composite preset logic (Stage 2).** For each symbol, check each group: group passes if ALL screeners in group pass (AND). Preset passes if ANY group passes (OR). Assign score as average of passing group scores. Include `matching_groups` in result. | per-screener results | composite result dict | Short-circuit evaluation: skip remaining groups if one passes | Draft |
| SRD-SCR-003.005 | FO-SCR-003 | Must | **Weighted preset logic (Stage 2).** For each symbol, sum weighted scores: Σ(score_i × weight_i) / Σ(weights). Symbol passes if combined score ≥ threshold. Include individual scores in details. | per-screener results, weights | composite result dict | Disabled screeners have weight 0 (excluded from sum) | Draft |
| SRD-SCR-003.006 | FO-SCR-003 | Must | **Stage 3: Optional LLM Ranking.** If preset includes LLM screener (screener_type="llm_claude" or "llm_local"), execute this stage. (5) Return top-N (e.g., top-5). | SRD-SCR-003.007 | FO-SCR-003 | Must | `ScreenerRunResult` dataclass: `preset_id`, `preset_type`, `run_timestamp` (ISO-8601), `execution_mode` (Literal["scheduled", "manual"]), `total_symbols_screened`, `symbols_after_prefilter`, `passed_count`, `results` (list of symbol result dicts), `llm_ranking` (optional). | execution output | result object | Serializable to JSON | Draft |
| SRD-SCR-003.008 | FO-SCR-003 | Should | Execution time targets: Stage 1 <1s, Stage 2 <30s, Stage 3 (if enabled) <2s. Total <2min for full S&P 500. Monitor and log times per stage for performance tuning. | timing checkpoints | log entries | Times are informational; no hard timeout enforced | Draft |
