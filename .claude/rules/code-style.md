# Code Style Rules

Extracted from `process.md ¬ß8` and `skill.md ¬ß1`. Always-on for every coding task.

## Linting & Type Checking

- Run `ruff check` before declaring any module done
- Run `mypy --strict` ‚Äî zero errors allowed
- Line length: 99 characters (`tool.ruff.line-length = 99`)
- Target: Python 3.11 (`target-version = "py311"`)

## Package Layout

- All production code lives under `us_swing/src/usswing/<tool>/`
- Importable as `from usswing.<tool> import ...`
- Shared cross-cutting code goes in `src/usswing/core/` ‚Äî never duplicate across tools
- No direct imports across tool boundaries (e.g. `screener` must not import from `analysis` directly; use `core/`)
- GUI code lives in `src/usswing/<tool>/gui/` ‚Äî no business logic in GUI modules
- MCP JSON Schemas live in `src/usswing/<tool>/schemas/`

## Code Orientation ‚Äî MODULE_MAP.json

Each source subfolder contains a `MODULE_MAP.json` index of all classes, method signatures, docstrings, and line numbers. It is generated by `us_swing/tools/skeleton_extractor/` and auto-refreshed via hook after every `.py` edit.

**Rules:**
- Never read `MODULE_MAP.json` directly ‚Äî use the query CLI via Bash
- Never read a full source file for orientation ‚Äî query first, then `Read()` only the specific section you need to edit
- Do not edit `MODULE_MAP.json` manually ‚Äî it is machine-generated

Use the **PowerShell tool** (not Bash) to run queries ‚Äî Bash cannot handle Windows paths reliably:

```powershell
$env:PYTHONPATH = "F:\USMarket_Backtesting\us_swing\tools"; python -m skeleton_extractor query --class <ClassName>
$env:PYTHONPATH = "F:\USMarket_Backtesting\us_swing\tools"; python -m skeleton_extractor query --overview <subfolder>
$env:PYTHONPATH = "F:\USMarket_Backtesting\us_swing\tools"; python -m skeleton_extractor query --find <symbol_substring>
```

## Function & Class Standards

- Type annotations required on all public functions and methods
- Google-style docstrings on all public APIs
- Use `dataclasses` or Pydantic models for data containers (not plain dicts)
- Prefer `Protocol` over ABCs for duck-typed interfaces
- Use `__slots__` on hot-path classes

## GUI Widget Sizing (PyQt6)

All interactive widgets must use the standard heights defined in `theme.C`:

| Widget type | Constant | Value |
|---|---|---|
| Any `QPushButton` | `C.BTN_H` | 28 px |
| `QLineEdit`, `QSpinBox`, `QDoubleSpinBox`, `QComboBox`, `QTimeEdit` | `C.INPUT_H` | 28 px |

**Rules:**
- Never hardcode a pixel height for a button or input ‚Äî use `C.BTN_H` / `C.INPUT_H`.
- The global QSS in `theme.py` enforces `min-height` + `max-height` for all variants. You only need `setFixedWidth()` in panel code ‚Äî **never `setFixedHeight()` or `setFixedSize()`** unless there is a documented exception.
- Never add `setStyleSheet("padding: ...")` to individual buttons to fix alignment ‚Äî fix the root cause in `theme.QSS` instead.
- `QPushButton#tab_btn` (title bar navigation) is the only exception: its height is `38 px` and is set in QSS.

## GUI Inline Stylesheet ‚Äî Focus Outline Rule (PyQt6)

Any `setStyleSheet()` call on a focusable widget (`QComboBox`, `QSpinBox`, `QDoubleSpinBox`, `QLineEdit`, `QPushButton`, `QRadioButton`) **must** include `outline: none` in both the base selector and the `:focus` pseudo-state. If no `:focus` override is needed, add it to the base rule only. Suppresses OS-drawn focus rectangle that clashes with custom border styling.

## GUI Window Style (PyQt6)

Every new top-level window or dialog **must** be frameless with a custom title bar. This is the mandatory pattern for all GUI work in this project.

**Setup:**
```python
self.setWindowFlags(Qt.WindowType.FramelessWindowHint | Qt.WindowType.Window)
```

**Custom title bar must include** (in order, right-aligned):
- Minimize button (`‚àí`) ‚Üí `window.showMinimized()`
- Maximize/restore button (`‚ñ°` / `‚ùê`) ‚Üí toggle via `showMaximized` / `showNormal`
- Close button (`‚úï`) ‚Üí `window.close()`

**Reference implementation:** `main_window.py::_TitleBar` (drag-to-move, maximize toggle, window-control buttons) and `scheduler_dialog.py::_DialogTitleBar` (dialog variant). Copy button style template from `_TitleBar.__init__` ‚Äî close hover `#c0392b`, others `C.OVERLAY2`.

**Titles and Headings:** Short noun phrase or imperative verb only ‚Äî no explanatory text. Omit words implicit from context. Bad: "Add New Trade". Good: "Add Trade".

## Key P0 Patterns (from skill.md ¬ß1)

- Decorators, context managers, generators ‚Äî prefer built-ins over custom when sufficient
- `asyncio` for all I/O-bound operations; never block the event loop
- Dependency injection over global state
- Repository pattern for all data access
