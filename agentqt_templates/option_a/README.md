# AgentQt вЂ” Agentic Waterfall Claude Code Framework for Python PyQt6 Projects

> A structured, agent-orchestrated development framework where AI agents gate every
> phase of a waterfall pipeline. Built for Python + PyQt6 projects using Claude Code.

---

## What Is AgentQt?

AgentQt gives your PyQt6 project a full software development lifecycle managed by
specialized Claude agents. Every feature follows a strict artifact chain:

```
FO в†’ SRD в†’ DD в†’ MD в†’ UTCD в†’ Code в†’ Tests в†’ RN
```

11 agents handle quality gates, code writing, review, and session management вЂ”
you only provide approvals at key checkpoints.

## How to Use This Template

### Step 1 вЂ” Create your repo from this template

Click **"Use this template"** on GitHub в†’ name your repo в†’ clone it locally.

### Step 2 вЂ” Create your project folder

Inside the cloned repo, create a folder named after your project:

```
mkdir <your_project_name>
```

Example: `mkdir stock_dashboard`

### Step 3 вЂ” Add your project definition files

Inside `<your_project_name>/`, create two files:

**`<your_project_name>/idea.md`** вЂ” Describe your project in plain English:
```
What is the project?
What problem does it solve?
Who uses it?
What are the main features at a high level?
```

**`<your_project_name>/requirements.md`** вЂ” List your tools/modules:
```
What are the main functional areas? (e.g. Screener, Dashboard, Settings)
For each area: what should it do?
Any technical constraints? (database, APIs, data sources)
Any non-functional requirements? (performance, offline support)
```

### Step 4 вЂ” Run /init

Open the repo in Claude Code and type:

```
/init
```

Claude will read your `idea.md` and `requirements.md` and generate:
- `CLAUDE.md` вЂ” project-specific instructions
- `AGENT_BOOT.md` вЂ” agent registry with your tool codes
- `<your_project_name>/CONTEXT.md` вЂ” current project state tracker
- `<your_project_name>/DEVLOG.md` вЂ” development log
- `<your_project_name>/docs/<tool>/` вЂ” one docs folder per tool

### Step 5 вЂ” Start building

Use the commands:

| Command | Purpose |
|---|---|
| `/new-feature` | Start a new feature (FO в†’ UTCD + code) |
| `/auto-feature` | Fully automated feature pipeline вЂ” no human gates |
| `/fix-issue` | Diagnose and fix a bug |
| `/refactor` | Refactor or enhance a module |
| `/write-tests` | Write pytest tests from UTCD |
| `/doc-check` | Audit all artifact docs for consistency |
| `/trace` | Sync TRACE.md traceability matrix |
| `/rn` | Write a Revision Note after implementation |

## Agent Overview

| Agent | Model | When It Runs |
|---|---|---|
| `prompt-evaluator` | Sonnet | Every Class D/S prompt вЂ” classifies and reframes |
| `duplicate-detector` | Haiku | Before writing any new artifact |
| `artifact-validator` | Haiku | After every FO/SRD/DD/MD/UTCD write |
| `phase-gate` | Haiku | Before starting code implementation |
| `session-finalizer` | Haiku | At the end of every command |
| `pyqt-architect` | Sonnet | GUI design decisions |
| `pyqt-code-writer` | Sonnet | Writing new PyQt6 files |
| `pyqt-code-reviewer` | Sonnet | Reviewing every PyQt6 file change |
| `pyqt-code-simplifier` | Sonnet | Only when reviewer flags complexity |
| `test-writer` | Sonnet | Writing pytest tests from UTCD |
| `code-reviewer` | Sonnet | Reviewing every non-GUI Python file |
