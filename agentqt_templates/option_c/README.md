# AgentQt — Agentic Waterfall Claude Code Framework for Python PyQt6 Projects

> A structured, agent-orchestrated development framework where AI agents gate every
> phase of a waterfall pipeline. Built for Python + PyQt6 projects using Claude Code.

---

## What Is AgentQt?

AgentQt gives your PyQt6 project a full software development lifecycle managed by
specialized Claude agents. Every feature follows a strict artifact chain:

```
FO → SRD → DD → MD → UTCD → Code → Tests → RN
```

11 agents handle quality gates, code writing, review, and session management —
you only provide approvals at key checkpoints.

## Quick Start (3 Steps)

### Step 1 — Clone this repo

```bash
git clone https://github.com/your-username/agentqt.git
cd agentqt
```

Or click **"Use this template"** on GitHub.

### Step 2 — Add your project definition

Create a folder named after your project and add two files inside it:

```
<your_project_name>/
├── idea.md          ← what your project is
└── requirements.md  ← what it needs to do
```

**`idea.md` format:**
```
Project name: ...
What it does: ...
Who uses it: ...
Main features (high level): ...
```

**`requirements.md` format:**
```
Tool / Module 1: <Name>
  - Requirement 1
  - Requirement 2

Tool / Module 2: <Name>
  - Requirement 1
  ...

Technical constraints: ...
Non-functional requirements: ...
```

### Step 3 — Run /init in Claude Code

Open the repo in Claude Code and type:

```
/init
```

Claude reads your two files and generates everything else automatically.
**You do not edit CLAUDE.md or AGENT_BOOT.md manually** — `/init` handles that.

---

## What /init Generates

From your `idea.md` and `requirements.md`:

| File | What It Contains |
|---|---|
| `CLAUDE.md` | Project instructions for Claude — prompt classification, session rules |
| `AGENT_BOOT.md` | Agent registry with your tool codes, routing decision tree |
| `<project>/CONTEXT.md` | Current project state tracker |
| `<project>/DEVLOG.md` | Development log (newest entry always at top) |
| `<project>/docs/<tool>/` | One docs folder per tool — FO, SRD, DD, MD, UTCD, TRACE stubs |

The `.claude/` directory (agents, commands, rules) is **pre-built** — no generation
needed. It is the same for every AgentQt project.

## Commands Reference

| Command | Phase | What It Does |
|---|---|---|
| `/init` | Setup | Reads idea.md + requirements.md, generates all project files |
| `/new-feature` | FO → Code | Full feature pipeline with human approval gates |
| `/auto-feature` | FO → RN | Fully automated, no gates — for well-defined features |
| `/fix-issue` | Bug → RN | Diagnose + fix + revision note |
| `/refactor` | Code | Safe refactor with post-review gate |
| `/write-tests` | UTCD → pytest | Write tests from UTCD document |
| `/doc-check` | Anytime | Read-only audit of all artifact docs |
| `/trace` | Anytime | Manual sync of TRACE.md |
| `/rn` | Post-impl | Write Revision Note after a feature or fix |

## Agent Overview

### Gate Agents — run on Haiku (fast, cheap)

| Agent | When |
|---|---|
| `prompt-evaluator` | Every Class D/S prompt — classifies and reframes before any action |
| `duplicate-detector` | Before writing any new artifact — returns existing anchor or GO |
| `artifact-validator` | After every FO/SRD/DD/MD/UTCD write — checks ID chains |
| `phase-gate` | Before code starts — verifies all SRDs Approved, UTCD complete |
| `session-finalizer` | End of every command — syncs TRACE, CONTEXT, DEVLOG |

### Implementation Agents — run on Sonnet

| Agent | When |
|---|---|
| `pyqt-architect` | Any GUI design decision or new panel/tool |
| `pyqt-code-writer` | Writing new PyQt6 files from an architect blueprint |
| `pyqt-code-reviewer` | After every PyQt6 file write or edit |
| `pyqt-code-simplifier` | Only when reviewer flags MEDIUM+ complexity |
| `test-writer` | When UTCD phase begins for a module |
| `code-reviewer` | After every non-GUI Python file write or edit |

## The Artifact Chain

Every feature follows this order — no skipping:

```
FO        Functional Objective      — what the feature achieves
SRD       Software Requirement      — specific requirements (Must/Should/Could)
DD        Design Document           — how it will be built
MD        Module Definition         — which files, classes, methods
UTCD      Unit Test Case Document   — test cases written before code
Code      Implementation            — only after UTCD exists
Tests     pytest                    — implements UTCD test cases
RN        Revision Note             — records what was built and why
```

Human approval required at: **SRD** (before design begins) and **FO** (before any work).
