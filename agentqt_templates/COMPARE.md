# AgentQt — Template Options Comparison

---

## Option A — GitHub Template Repo (Repo only, /init does the work)

**File:** `option_a/README.md`

**User steps:**
1. Click "Use this template" on GitHub
2. Create `<project_name>/idea.md` and `<project_name>/requirements.md`
3. Open in Claude Code
4. Run `/init`
5. Start building

**What's pre-built in the repo:**
- All 11 agents (`.claude/agents/`)
- All commands including `/init` (`.claude/commands/`)
- All rules (`.claude/rules/`)

**What `/init` generates:**
- `CLAUDE.md`, `AGENT_BOOT.md`, `CONTEXT.md`, `DEVLOG.md`, docs folder per tool

**Pros:**
- Clean GitHub experience — one-click template
- Users get updates if they watch the repo
- Easy to contribute back improvements
- No giant prompt to paste

**Cons:**
- Requires GitHub account and git knowledge
- User must know how to use Claude Code slash commands

---

## Option B — Single Bootstrap Prompt (No repo needed)

**File:** `option_b/agentqt_bootstrap.md`

**User steps:**
1. Download one file: `agentqt_bootstrap.md`
2. Create `<project_name>/idea.md` and `<project_name>/requirements.md`
3. Open project folder in Claude Code
4. Paste the entire bootstrap file as first message
5. Claude generates everything including `.claude/` directory

**What the bootstrap prompt does:**
- Generates ALL files from scratch — agents, commands, rules, CLAUDE.md, AGENT_BOOT.md, docs
- Self-contained — no prior repo structure needed

**Pros:**
- Zero setup — just paste and go
- Works without GitHub
- One file to share (Discord, email, Gist)

**Cons:**
- Generates agent/command/rule files every time (redundant if shared)
- 50,000+ token prompt — expensive first call
- Hard to update — users won't get improvements automatically
- Agent files generated by AI may drift from canonical versions

---

## Option C — Both: Repo + /init Command (Recommended)

**File:** `option_c/README.md`

**User steps:**
1. Clone repo (or click "Use this template")
2. Create `<project_name>/idea.md` and `<project_name>/requirements.md`
3. Open in Claude Code
4. Run `/init`
5. Start building

**Split of work:**
- Repo carries: agents, commands, rules (framework — never changes per project)
- `/init` generates: CLAUDE.md, AGENT_BOOT.md, CONTEXT.md, DEVLOG.md, docs structure (project-specific)

**Pros:**
- Best of both worlds
- Agent/command/rule files are canonical — version controlled in one place
- `/init` only generates the small project-specific files (cheap call)
- Users can pull framework updates without touching their project files
- README serves as full documentation

**Cons:**
- Requires git / GitHub
- Slightly more steps than Option B on first use

---

## Recommendation

| If your users are... | Go with |
|---|---|
| Developers comfortable with GitHub + Claude Code | Option A or C |
| Non-technical or want zero-friction sharing | Option B |
| You want the most maintainable, scalable solution | **Option C** |

**My pick: Option C** — the repo is the framework (stable, versioned), `/init` is
the personalizer (runs once, generates project-specific files). Clean separation,
easy to update, easy to document.
